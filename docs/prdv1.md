**产品名称**：私人投资 AI 助理（一期）  
**版本**：v1.0  
**文档日期**：2026.04  
**文档状态**：已最终确认，可直接用于开发  

## 1. 产品概述

**定位**：个人专属、多Agent驱动的**金融分析认知增强工具**（一期专注美股）。  
**核心价值**：
- 通过大模型 + 多Agent能力，帮助用户**积累金融分析技能**，而非简单给出答案。
- 解决“看盘 / 盯盘”核心痛点：自动解读新闻、SEC、基本面、技术指标，让用户真正看懂。
- **完全私密**：持仓数据 100% 本地存储，不上传任何云端。

**目标用户**：有一定投资经验、希望自己决策但信息碎片化严重的个人投资者（美股为主）。

**一期范围限制**（严格控制复杂度）：
- 仅支持美股。
- 不做扫描选股器（仅支持手动添加股票代码）。
- 持仓录入极简（仅数量 + 成本价）。
- 金融数据仅接 **Finnhub**，其余全部靠大模型能力获取。
- 无实时行情推送需求。

## 2. 大目标 & 具体目的

**大目标**：积累个人金融 skill，通过大模型能力增强个人认知，降低获取信息难度。

**具体目的（一期）**：
1. 每日/手动完成**持仓全景分析 + 个股深度分析**。
2. 通过多Agent协作 + **结果融合**，输出可学习、可信任的分析报告。

## 3. 核心功能模块

### 3.1 持仓管理（极简版）
- 使用本地 JSON/YAML 文件存储持仓列表。
- 每条记录字段（必须）：
  - `ticker`（美股代码，如 AAPL、NVDA）
  - `quantity`（持仓数量）
  - `cost_price`（成本价）
  - `note`（可选，个人备注 / 投资风格 / 关注点）
- 首页仪表盘：显示总市值、总盈亏、Agent综合评分（饼图 + 列表）。
- 支持单股票点击进入**深度分析页**。
- 支持批量手动选择股票触发分析。

### 3.2 分析引擎（产品核心）
**触发方式**：手动点击“今日分析”或“分析选中股票”（一期不做自动定时）。

**多Agent协作流程**：
1. **Supervisor Agent**（总控）：读取用户全局投资风格 + ticker专属note，拆解任务并下发。
2. **专项Agent**（共5个，可配置）：
   - **News Agent**：最新新闻 + 用户自定义关键事件（权威来源）。
   - **SEC Agent**：EDGAR 表格变化（Form 4、13F、10-K 等）。
   - **Fundamentals Agent**：基本面（Finnhub 数据 + 大模型补齐）。
   - **Technical Agent**：技术指标自动计算并解读（MACD、RSI、布林带等，用户无需懂）。
   - **Custom Skill Agent**：用户可随时安装自定义 skill。
3. **Fusion Agent（必做）**：
   - 接收所有专项Agent输出。
   - **平衡型融合逻辑**（用户指定B）：
     - 加权综合结论（标明各Agent权重/置信度）。
     - 明确标注分歧点。
     - 给出风险提示。
     - 最终输出“可行动建议”（非激进买卖指令，而是供用户决策参考）。

**输出格式**：
- 最上方：Fusion Agent 融合总结（醒目）。
- 下面：每个Agent独立报告（可点击展开完整推理链）。
- 采用 Markdown + 简单图表（Recharts）。

**数据获取规则**：
- 仅使用 **Finnhub** 获取股价、财报、技术指标等结构化数据。
- 新闻、SEC解读、市场上下文、事件影响等**全部靠大模型能力**（不额外接实时API）。

### 3.3 个性化能力
- 全局**投资风格上下文**（用户一次性输入或随时编辑）。
- 每个ticker可单独绑定关注点（prompt里直接注入）。
- 新闻追踪支持用户自定义关键词 + 权威维度（prompt可改）。

### 3.4 Agent & 模型配置
- 支持给任意Agent安装/卸载自定义 skill。
- 支持**单独与单个Agent聊天**（调试、测试 skill、修改 prompt）。
- 单聊界面可实时切换模型配置。
- 模型接入支持：
  - OpenRouter
  - 原生 Grok / Gemini / 阿里百炼
  - 任意 OpenAI 兼容接口

## 4. 非功能需求
- **私密性**：持仓文件 + API Key 全部本地存储。
- **技术选型**：
  - 后端：AgentScope（多Agent框架）。
  - 前端：React（推荐 Next.js + Tailwind + shadcn/ui + Recharts）。
  - 存储：本地 JSON（后续可平滑升级为 SQLite）。
- **复杂度控制**：一期只做上述范围，后续再迭代。
