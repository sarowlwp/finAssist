# Markdown 报告渲染系统设计

## 概述

本设计旨在改进金融分析报告的阅读体验，使用功能强大的 Markdown 渲染系统替代当前简单的渲染实现。

## 目标

- 提供完整的 Markdown 语法支持
- 优化阅读布局和视觉层次
- 支持代码高亮、表格、链接等增强功能
- 保持良好的响应式和夜间模式支持
- 添加目录导航功能

## 技术方案

### 核心库选择

采用 **方案1：react-markdown** 作为核心渲染库

**选择理由：**
- 成熟稳定，社区活跃
- 支持完整的 CommonMark 和 GFM 规范
- 可通过插件系统扩展功能
- 与 React 生态系统良好集成

### 技术栈

| 组件 | 库/工具 | 用途 |
|------|---------|------|
| Markdown 渲染 | react-markdown | 核心渲染引擎 |
| GFM 支持 | remark-gfm | GitHub Flavored Markdown |
| 代码高亮 | react-syntax-highlighter | 代码块语法高亮 |
| 样式系统 | Tailwind CSS | 响应式布局和主题 |
| UI 组件 | shadcn/ui | 卡片、按钮等 UI 元素 |

## 功能设计

### 1. Markdown 语法支持

#### 基础格式
- 标题（H1-H6）
- 段落和换行
- 粗体和斜体
- 删除线
- 水平线

#### 列表
- 有序列表
- 无序列表
- 任务列表（GFM）
- 嵌套列表

#### 链接和图片
- 内联链接
- 引用链接
- 自动链接
- 图片渲染（可点击放大）

#### 代码
- 行内代码
- 代码块（支持语法高亮）
- 支持语言：JSON、Python、JavaScript、TypeScript、SQL、Bash

#### 表格（GFM）
- 表头和表体
- 对齐方式（左、中、右）
- 金融数据优化显示

#### 引用
- 块引用
- 嵌套引用

### 2. 增强功能

#### 目录导航
- 自动从标题生成目录
- 侧边栏固定显示
- 点击跳转到对应章节
- 当前章节高亮

#### 链接优化
- 外部链接：新窗口打开，显示外部链接图标
- 内部链接：平滑滚动
- 安全属性：rel="noopener noreferrer"

#### 折叠区域
- 支持 `<details>` 和 `<summary>` 标签
- 或使用自定义折叠组件
- 适用于长段落或详细数据

#### 阅读模式
- 优化的行高（1.6-1.8）
- 合适的字体大小
- 最大宽度限制（便于阅读）
- 适当的段落间距

### 3. 布局设计

#### 整体结构
```
┌─────────────────────────────────────────────────┐
│  顶部导航栏（固定）                                │
├──────────────────┬──────────────────────────────┤
│                  │  Fusion 摘要卡片               │
│  目录侧边栏      ├──────────────────────────────┤
│  (可折叠)        │  Agent 报告卡片 1            │
│                  ├──────────────────────────────┤
│                  │  Agent 报告卡片 2            │
│                  ├──────────────────────────────┤
│                  │  ...                          │
└──────────────────┴──────────────────────────────┘
```

#### 响应式设计
- **桌面（>1024px）**：侧边栏目录 + 主内容区
- **平板（768-1024px）**：可折叠侧边栏
- **移动（<768px）**：顶部目录下拉菜单

#### Agent 报告卡片
- 每个 Agent 报告独立卡片
- 标题栏：Agent 名称、表情符号、状态
- 可折叠/展开
- 边框颜色区分不同 Agent

### 4. 主题支持

#### 浅色模式
- 背景：白色/浅灰
- 文字：深灰/黑色
- 代码块：浅色背景

#### 深色模式
- 背景：深蓝灰/黑色
- 文字：浅灰/白色
- 代码块：深色背景（Monokai 风格）

## 风险和注意事项

### 性能考虑
- 大文件渲染：使用虚拟滚动或分段渲染
- 代码高亮：避免过多语言支持
- 目录生成：缓存解析结果

### 兼容性
- 保持向后兼容现有数据格式
- 处理不规范的 Markdown 输入
- 移动端性能优化

### 安全性
- 防止 XSS 攻击（react-markdown 默认处理）
- 外部链接安全属性
- 图片内容验证
