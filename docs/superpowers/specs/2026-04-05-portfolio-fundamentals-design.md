# 持仓页面基本面数据展示设计

## 概述

为金融分析 AI 系统的持仓页面添加股票常用的基本面金融数据展示和管理功能，方便用户在一个页面内完成持仓管理和股票分析。

## 目标用户

金融投资者、分析师，需要在一个页面内查看和管理持仓，同时获取关键的股票基本面数据。

## 功能需求

### 1. 数据展示

- **公司概况**：公司名称、行业、市值、描述
- **估值指标**：市盈率(PE)、市净率(PB)、市销率(PS)、股息率
- **盈利能力**：净资产收益率(ROE)、资产收益率(ROA)、净利润率、营收增长率
- **财务健康**：负债率、流动比率、速动比率
- **技术指标**：RSI、MACD、布林带（可选）

### 2. 数据管理

- **SQLite 缓存**：所有数据必须经过缓存，避免频繁API调用
- **手动更新**：支持用户手动触发数据刷新
- **数据过期提示**：显示数据更新时间和过期状态

### 3. 用户体验

- 保持与现有界面风格一致
- 支持浅色/深色模式
- 响应式设计，适配移动设备
- 加载状态和错误处理

## 技术架构

### 1. 后端架构

#### 数据获取层

- 复用 `FinnhubService` 类，使用现有的 API 方法：
  - `get_company_profile()`：获取公司概况
  - `get_financials()`：获取财务指标
  - `get_technical_indicators()`：获取技术指标

#### 缓存层

- 复用 `FinnhubCache` 数据库表
- 复用 `FinnhubCacheService` 缓存管理服务
- 不同数据类型使用不同的过期策略：
  - 公司概况：24小时
  - 财务指标：12小时
  - 技术指标：6小时

#### API 接口

- 扩展现有 `/portfolio` 接口，返回持仓信息时包含数据更新状态
- 新增 `/portfolio/{ticker}/fundamentals` 接口，获取单只股票的完整基本面数据
- 新增 `/portfolio/{ticker}/fundamentals/refresh` 接口，强制刷新数据

### 2. 前端架构

#### 组件设计

- **PortfolioTable**：增强现有表格，添加数据查看按钮
- **FundamentalsModal**：基本面数据详情模态框
- **DataRefreshButton**：数据刷新组件
- **LoadingState**：加载状态组件
- **ErrorState**：错误处理组件

#### 数据管理

- 使用现有 API 调用方式
- 添加数据刷新状态管理
- 支持缓存数据和新鲜数据的显示

#### 样式设计

- 复用现有 Tailwind CSS 配置
- 使用现有的配色方案和组件风格
- 确保支持浅色/深色模式

## 风险管理

### 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| API 调用失败 | 高 | 用户无法获取数据 | 实现优雅降级，显示缓存数据 |
| 数据格式变更 | 中 | 显示数据错误 | 使用类型检查和默认值 |
| 性能问题 | 中 | 页面加载慢 | 优化 API 调用，实现懒加载 |

### 用户体验风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 数据过载 | 中 | 界面混乱 | 只显示核心指标，提供查看更多选项 |
| 刷新延迟 | 中 | 用户等待 | 显示进度提示，提供取消选项 |

## 成功标准

1. 所有设计的功能正常工作
2. 数据准确并及时更新
3. 界面响应迅速，加载时间 < 2秒
4. 支持浅色/深色模式
5. 用户可以直观地找到和使用功能
