### 技术背景
这是一个 python + react 的开发的金融分析Agent应用代码仓库。

### 用户习惯
1、每次 compact 后，提交本地 git 并 push 到远端，重启服务后告知用户体验
2、尽可能使用 subagent 加速构建

### 设计规范
1. 需要保持金融端的界面风格，考虑屏幕使用效率。

### 开发规范
1. 使用统一的变量来定义样式，避免定制，修改兼容考虑夜间模式和白天模式。
2. 修改必须考虑 backend、frontend 两部分，不能顾头不顾尾
3. 修正 Agent 同步分析接口，同时需要考虑 SSE 方式接口正确。
4. 添加外部系统的关键输入输出日志，排查问题。
5. 开发完毕重启前端和后端服务，保证代码最新

### 测试规范
1. 所有页面功能必须覆盖完整的E2E测试，修改后使用Chrome浏览器验证，严格遵守 `.claude/E2E_TESTS.md` 文件
2. 功能完成必须有测试，测试必须覆盖新增功能，有测试必须执行，执行结果来判定。
3. 所有修改必须要单元测试
4. 发现测试问题必须修复，保证测试正确性。
5. 测试需要考虑样式、功能、页面展示、性能正确。

### 服务重启
- 后端服务: http://localhost:8001
- 前端服务: http://localhost:3000
- 详细重启方法请参考 README.md 中的 "服务重启方法" 章节
- 重启前确保停止旧服务进程，避免端口冲突

### 教训记录（2026-04-04）
1. 样式修改时先查看 git 原始版本，保持项目原有样式系统，功能改进和样式修改要分开。
2. 修改页面内容后要同步检查和更新测试文件，特别注意硬编码的端口和内容匹配问题。
3. 不要让卡片在没有内容时使用 hidden 类，保持所有卡片可见并显示适当的提示文字。
